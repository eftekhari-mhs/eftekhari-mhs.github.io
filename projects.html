I am working on population protocols, an appropriate model for electronic computing scenarios such as sensor networks and for ''fast-mixing'' physical systems such as chemical reactions, gene regulatory networks, and animal populations.

Population protocols are a complete network of agents without any control of whom they're going to interact with. A random scheduler selects a pair of agents and they interact based on the designed protocol and their current states. Because of the applications of population protocols it is important that agents require small number of states.

Uniformity in Population Protocols

A model of computation describe how to compute the outputs, given the inputs by illustrating how to operate the memory and communications of the system. To have a general model, we define a ''uniform'' variation of population protocol in which describes how a set of outputs are computed given a set of inputs using each agent's local algorithm for computation without any prior knowledge of population size. Such an algorithm may produce outputs longer than its inputs, retaining the ability to use a number of states that grows with the population size. Exact/Approximate Counting

The problem of counting the number of agents in a population. Many of existing protocols assume the agents know what is the population size. This is a strong assumption and in our works we focused on the protocols that compute the population size.
